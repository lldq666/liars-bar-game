## 🎮 游戏简介

**骗子酒馆**是一款心理博弈卡牌游戏。玩家需要轮流打出卡牌并声称打出的是什么牌，其他玩家可以选择相信或质疑。谎言被揭穿将付出代价，而成功的欺骗则能让对手陷入困境！

### 🎯 游戏目标

成为最后一个拥有筹码的玩家，在心理博弈中战胜所有对手！

---

## ✨ 功能特性

### 核心功能

- ✅ **完整的游戏逻辑** - 创建、加入、开始、出牌、质疑
- ✅ **多玩家支持** - 支持 2-4 名玩家同时游戏
- ✅ **准备系统** - 所有玩家准备后自动开始游戏
- ✅ **实时状态更新** - 2秒轮询间隔，流畅的游戏体验
- ✅ **胜利提示** - 游戏结束时显示胜利者和最终统计

### 高级功能

- 🔄 **多后端服务器切换** - 无缝切换不同后端服务器
- 🚀 **负载均衡支持** - 可同时运行多个后端实例
- 💾 **配置持久化** - 使用 localStorage 保存服务器配置
- 🎨 **响应式设计** - 支持桌面端和移动端

### 房间管理

- 🚪 **玩家离开功能** - 优雅地离开游戏并清理状态
- 🗑️ **自动释放房间** - 所有玩家离开后自动释放房间ID
- 🔄 **手牌自动刷新** - 新一轮开始时所有玩家手牌同步更新

## 🚀 快速开始

### 方式一：使用 Python 启动器 (最推荐)

```bash
# Windows / Linux / Mac 通用
python launcher.py
```

启动器会：

- ✓ 自动检查 Python 环境
- ✓ 自动安装缺失的依赖
- ✓ 提供友好的菜单界面
- ✓ 引导你完成启动

### 方式二：手动启动 (最可靠)

**步骤 1 - 安装依赖:**

```bash
pip install flask flask-cors
```

**步骤 2 - 启动后端服务器:**

```bash
cd backend
python server.py 5000
```

**步骤 3 - 启动前端 (新开一个命令行窗口):**

```bash
cd frontend
python -m http.server 8080
```

**步骤 4 - 打开游戏:**

- 浏览器访问: `http://localhost:8080`
- 或直接用浏览器打开: `frontend/index.html`

**步骤 5 - 配置服务器:**

1. 在游戏界面输入服务器地址: `http://localhost:5000`
2. 点击"测试连接"
3. 看到"连接成功"后点击"保存配置"
4. 点击"进入游戏大厅"

## 📘 详细使用指南

### 1. 创建游戏

1. 点击"创建游戏"按钮
2. 游戏ID可以留空（自动生成）
3. 选择最大玩家数（2-4人）
4. 点击"创建"

### 2. 加入游戏

1. 点击"加入游戏"按钮
2. 输入游戏ID
3. 输入你的玩家ID（如: player1）
4. 点击"加入"

### 3. 准备开始

1. 等待其他玩家加入
2. 每个玩家点击"准备"按钮
3. 准备状态实时显示（✅ 表示已准备，⏳ 表示未准备）
4. **当所有玩家都准备后，游戏自动开始！**

### 4. 游戏进行

#### 出牌阶段:

- 选择要出的牌（点击手牌）
- 选择声称的牌型
- 点击"确认出牌"

#### 质疑阶段:

- 如果怀疑上家说谎，点击"质疑!"
- 如果质疑成功 → 说谎者失去1个筹码
- 如果质疑失败 → 质疑者失去1个筹码

### 5. 游戏结束

- 当有玩家筹码为0时淘汰
- 最后剩下的玩家获胜
- 显示胜利提示和最终统计
- 点击"结束游戏并释放房间"清理资源

## 🔧 API 接口文档

### 后端 API 端点

| 方法   | 路径                             | 说明        |
| ---- | ------------------------------ | --------- |
| POST | `/api/game/create`             | 创建新游戏     |
| POST | `/api/game/{id}/join`          | 加入游戏      |
| POST | `/api/game/{id}/ready`         | 玩家准备/取消准备 |
| POST | `/api/game/{id}/start`         | 开始游戏      |
| POST | `/api/game/{id}/play`          | 出牌        |
| POST | `/api/game/{id}/challenge`     | 质疑        |
| POST | `/api/game/{id}/leave`         | 离开游戏      |
| POST | `/api/game/{id}/release`       | 释放游戏房间    |
| GET  | `/api/game/{id}/state`         | 获取游戏状态    |
| GET  | `/api/game/{id}/hand/{player}` | 获取玩家手牌    |
| GET  | `/api/config`                  | 获取服务器配置   |

### 请求示例

**创建游戏:**

```bash
curl -X POST http://localhost:5000/api/game/create \
  -H "Content-Type: application/json" \
  -d '{"game_id": "my_game", "max_players": 4}'
```

**加入游戏:**

```bash
curl -X POST http://localhost:5000/api/game/my_game/join \
  -H "Content-Type: application/json" \
  -d '{"player_id": "player1"}'
```

**出牌:**

```bash
curl -X POST http://localhost:5000/api/game/my_game/play \
  -H "Content-Type: application/json" \
  -d '{"player_id": "player1", "card_indices": [0, 1], "claimed_type": "K"}'
```

## 🃏 游戏规则

### 基本规则

1. **初始状态**
   
   - 每个玩家获得等量手牌
   - 每个玩家获得 10 个筹码

2. **游戏流程**
   
   - 玩家轮流打出 1 张或多张手牌
   - 打牌时必须**声称**打出的是什么牌（可以说谎！）
   - 下家可以选择：
     - **相信** → 继续游戏，轮到下一位玩家
     - **质疑** → 揭穿上家的谎言

3. **质疑结果**
   
   - 质疑成功（上家说谎）→ 说谎者失去 1 个筹码
   - 质疑失败（上家说真话）→ 质疑者失去 1 个筹码

4. **游戏结束**
   
   - 当玩家筹码为 0 时淘汰
   - 最后剩下的玩家获胜

### 💡 策略提示

- 可以适当说谎，但别被抓住!
- 观察对手的出牌模式和声称习惯
- 在关键时刻使用质疑
- 保持筹码数量，避免过早淘汰
- 有时说真话比说谎更能迷惑对手

## 🆕 最近更新

### ✅ 修复手牌刷新问题 (2026-06-20)

**问题描述:**
当一个玩家手牌出完后，后端正确收回所有玩家的牌并重新发牌，但前端的其他玩家没有刷新手牌显示。

**解决方案:**

- 添加轮次变化检测机制
- 在 `updateGameState()` 中检测 `round` 变化
- 当检测到新一轮开始时，自动调用 `loadHand()` 刷新手牌

**效果:**

- ✅ 所有玩家都能在重新发牌后及时看到自己的新手牌
- ✅ 手牌显示与游戏状态保持同步

### ✅ 修复玩家离开和房间自动释放问题 (2026-06-20)

**问题 1: 玩家退出后，玩家列表还显示该玩家**

- 实现 `/api/game/<game_id>/leave` API 端点
- 前端 `leaveGame()` 函数调用后端 API 通知服务器
- 正确清理玩家相关的所有状态

**问题 2: 所有玩家离开后，房间ID不释放**

- 在玩家离开时检测房间是否为空
- 如果房间为空且游戏未开始，自动删除房间
- 返回 `room_released: True` 通知前端

**效果:**

- ✅ 玩家离开后能从玩家列表中移除
- ✅ 玩家可以再次加入同一游戏
- ✅ 空房间自动释放，ID 可重用

### ✅ 添加胜利提示和房间释放功能

**新增功能:**

- 游戏结束时显示胜利提示弹窗
- 展示胜利者信息和最终筹码统计
- 添加"结束游戏并释放房间"按钮
- 释放房间ID供其他玩家重用

### ✅ 实现准备按钮系统

**改进:**

- 将"开始游戏"按钮改为"准备"按钮
- 所有玩家准备后游戏自动开始

### ✅ 修复质疑机制问题 (2026-06-20)

**问题描述:**
当玩家 3 出牌后，按理应该只有玩家 1 能够质疑（因为回合顺序是 1→2→3→1→...），但实际上是玩家 2 也能质疑。

**问题原因:**
前端在 `updateGameState()` 函数中，质疑按钮的显示逻辑有错误。原来的代码让**任何没有打出上一张牌的玩家**都能看到质疑按钮，但实际上**只有当前回合的玩家**才能质疑。

**解决方案:**

- 统一前端的状态检测逻辑
- 优先判断是否是当前玩家的回合（`isMyTurn`）
- 只有在当前玩家回合的情况下，才根据 `last_claim` 决定是否显示质疑按钮
- 非当前回合的玩家完全看不到任何操作按钮
- 修复 `updateLastPlay()` 函数，让它只更新内容，不控制显示/隐藏

**效果:**

- ✅ 只有当前回合的玩家才能看到质疑按钮
- ✅ 其他玩家看不到质疑按钮和出牌按钮
- ✅ 质疑机制符合游戏规则
- ✅ 修复了 `updateLastPlay()` 和 `updateGameState()` 之间的逻辑冲突

**修改文件:**
- `frontend/game.js` - 修改 `updateGameState()` 和 `updateLastPlay()` 函数

### ✅ 添加调试信息 (2026-06-20)

**改进:**

- 在 `updateGameState()` 中添加详细的控制台日志
- 输出 `current_player`、`playerId`、`isMyTurn`、`last_claim` 等信息
- 方便调试质疑按钮显示问题

## ❓ 常见问题

### Q1: 无法连接服务器

**A:** 检查以下问题：

- 后端服务器是否启动？
- 端口号是否正确（默认 5000）？
- 防火墙是否允许该端口？
- 服务器地址是否输入正确？

### Q2: 跨域错误 (CORS)

**A:** 确保后端已安装 flask-cors：

```bash
pip install flask-cors
```

### Q3: 游戏状态不更新

**A:** 尝试以下操作：

- 刷新页面重新连接
- 检查游戏ID和玩家ID是否正确
- 查看浏览器控制台是否有错误
- 确认后端服务器是否正常运行

### Q4: 端口已被占用

**A:** 更换端口号：

```bash
# 后端使用 5001 端口
python backend/server.py 5001

# 前端使用 8081 端口
cd frontend
python -m http.server 8081
```

然后在游戏界面的"服务器配置"中修改服务器地址。

### Q5: 如何与他人联机?

**A:** 需要以下步骤：

1. 服务器需要公网IP或使用内网穿透工具（如 ngrok）
2. 其他玩家在"服务器配置"中输入你的服务器地址
3. 确保所有玩家连接到同一个服务器和游戏ID
