## 1. 游戏简介

### 🎯 什么是骗子酒馆？

骗子酒馆（Liar's Bar）是一款**心理博弈卡牌游戏**。在游戏中，你需要：

- 🃏 **打出卡牌** - 但你可以选择说真话或说谎话
- 🤔 **判断对手** - 对手说的是真还是假？
- 💥 **质疑对手** - 如果你认为对手在说谎，可以揭穿他们
- 🎭 **演技比拼** - 成功地欺骗对手，让他们付出代价

### 🏆 游戏目标

成为最后一个拥有筹码的玩家！

当其他所有玩家都输光筹码时，你就赢了！

### 💡 为什么叫"骗子酒馆"？

因为在游戏中，**说谎是合法的策略**！你经常需要声称自己打出的是某种牌，但实际上可能是另一种。成功地欺骗对手，让他们相信你的谎言，这就是游戏的乐趣所在！

## 2. 第一次启动游戏

### 🚀 快速启动（3分钟上手）

#### 步骤 1: 安装依赖

打开命令行（终端），进入游戏文件夹，运行：

```bash
pip install flask flask-cors
```

> **提示：** 如果你已经安装过，可以跳过这一步。

#### 步骤 2: 启动后端服务器

**最简单的方法 - 使用启动器：**

```bash
python launcher.py
```

然后选择 "1. 仅启动后端服务器"

> **或者手动启动：**
> 
> ```bash
> cd backend
> python server.py 5000
> ```

你应该看到类似这样的输出：

```
* Running on all addresses (0.0.0.0)
* Running on http://127.0.0.1:5000
* Running on http://192.168.x.x:5000
```

#### 步骤 3: 启动前端

**新开一个命令行窗口**，运行：

```bash
cd frontend
python -m http.server 8080
```

应该看到：

```
Serving HTTP on 0.0.0.0 port 8080 ...
```

#### 步骤 4: 打开游戏

在浏览器中访问：`http://localhost:8080`

#### 步骤 5: 配置服务器

第一次打开游戏时，你会看到"服务器配置"界面：

1. 输入服务器地址：`http://localhost:5000`
2. 点击"连接"按钮
3. 点击"进入游戏大厅"

> **💡 小贴士：** 如果你之后想切换服务器，可以随时点击界面上的"服务器配置"按钮。

## 4. 基本游戏规则

### 🃏 卡牌说明

游戏使用标准扑克牌（不含大小王），每种牌有 4 张：

| 牌型   | 说明       |
| ---- | -------- |
| A    | 最小的牌     |
| 2-10 | 数字牌      |
| J    | 骑士       |
| Q    | 皇后       |
| K    | 国王（最大的牌） |

> **💡 小贴士：** 虽然牌型有大小之分，但在本游戏中，**牌型的大小不影响游戏结果**，只是用来"声称"的。

### 🎯 游戏流程

让我们用最简单的方式理解游戏流程：

#### 第 1 步：发牌和初始筹码

- 每个玩家获得 **等量手牌**（具体数量取决于玩家人数）
- 每个玩家获得 **10 个筹码**

#### 第 2 步：轮流行动

游戏按顺时针方向轮流进行，每个回合包含：

1. **上家打出卡牌**（并声称是什么牌）
2. **你选择**：
   - **相信** → 轮到你时，你继续打出卡牌
   - **质疑** → 揭穿上家的谎言

#### 第 3 步：质疑的结果

当你选择质疑时：

- ✅ **质疑成功**（上家确实在说谎）
  
  - 上家失去 **1 个筹码**
  - 轮到你打出卡牌

- ❌ **质疑失败**（上家说的是真话）
  
  - 你失去 **1 个筹码**
  - 轮到你打出卡牌

#### 第 4 步：重新开始

当有玩家打完所有手牌时：

- 所有玩家的牌都被收回
- 重新洗牌并发牌
- 开始新一轮

> **💡 小贴士：** 这保证了所有玩家始终有牌可打！

#### 第 5 步：游戏结束

- 当有玩家的筹码数降为 **0** 时，该玩家被淘汰
- 最后剩下的玩家（即有筹码的玩家）**获胜**！

## 5. 完整游戏流程

让我们通过一个完整的例子来理解游戏！

### 🎬 场景：3 人游戏

**玩家：** 小明、小红、小刚
**初始状态：** 每人 10 个筹码，每人 5 张手牌

---

### 回合 1：小明的回合

**小明手牌：** A, A, 2, 3, K

小明想打出 2 张 A，但他决定**说谎**，声称打出了 2 张 K：

1. 小明选择 2 张手牌（实际是 A）
2. 在"声称的牌型"中选择 "K"
3. 点击"确认出牌"

**游戏日志显示：**

```
小明 声称打出了 2张 K
```

---

### 回合 2：小红的回合

**小红的抉择：**

小红心想："小明声称打出了 K，但他平时不会这么早打出大牌...我怀疑他在说谎！"

小红有两个选择：

**选择 A - 相信小明：**

- 小红不打出牌，直接跳过（在本版本中，相信意味着轮到下家）
- 轮到小刚行动

**选择 B - 质疑小明：**

- 小红点击"质疑!"按钮
- 系统检查小明是否说谎

让我们看看两种结果：

#### 结果 1：小红质疑，成功了！

系统揭示：小明实际打出的是 A，不是 K！

**游戏日志显示：**

```
小红 选择质疑
质疑成功！小明 失去1个筹码
```

**结果：**

- 小明失去 1 个筹码（还剩 9 个）
- 轮到小红打出卡牌

#### 结果 2：小红质疑，失败了！

系统揭示：小明实际打出的确实是 K！

**游戏日志显示：**

```
小红 选择质疑
质疑失败！小红 失去1个筹码
```

**结果：**

- 小红失去 1 个筹码（还剩 9 个）
- 轮到小红打出卡牌

---

### 回合 3：继续游戏

游戏继续进行，直到：

1. 有玩家筹码降为 0 → 淘汰
2. 最后只剩一个玩家 → 获胜！

## 6. 新手入门教程

### 🎓 第一步：创建你的第一场游戏

1. 启动游戏，进入游戏大厅
2. 点击"创建游戏"
3. 游戏 ID 可以留空（会自动生成，如 "game_abc123"）
4. 选择最大玩家数（建议选择 2 或 3，便于学习）
5. 点击"创建"

**你应该看到：**

```
✅ 游戏创建成功！
游戏ID: game_abc123
等待其他玩家加入...
```

### 🎓 第二步：加入游戏（自己测试）

如果你想自己测试游戏机制：

1. **打开新的浏览器窗口或标签页**（模拟第二个玩家）
2. 访问 `http://localhost:8080`
3. 配置相同的服务器地址（`http://localhost:5000`）
4. 点击"加入游戏"
5. 输入相同的游戏 ID（`game_abc123`）
6. 输入不同的玩家 ID（如 "player2"）
7. 点击"加入"

> **💡 小贴士：** 你可以用不同的浏览器（Chrome、Firefox、Edge）或同一浏览器的不同窗口来模拟多个玩家。

### 🎓 第三步：准备开始

1. 在每个玩家窗口中，点击"准备"按钮
2. 观察准备状态：
   - 你已经准备的窗口：按钮变成"取消准备"
   - 其他玩家窗口：你的状态显示为 "✅"
3. 当所有玩家都准备后，游戏会自动开始！

**你应该看到：**

```
✅ 所有玩家已准备！
游戏开始！第 1 轮
```

### 🎓 第四步：打出你的第一张牌

1. **选择手牌** - 点击你想打出的牌（会高亮显示）
2. **选择声称的牌型** - 从下拉菜单中选择
3. **确认出牌** - 点击"确认出牌"按钮

> **⚠️ 重要：** 你选择的声称牌型**可以是真话，也可以是谎话**！

### 🎓 第五步：学会质疑

当轮到你的回合，且上家刚刚打出牌时：

1. 阅读游戏日志，看看上家声称打出了什么牌
2. 判断上家是否在说谎
3. 如果你想质疑：
   - 点击"质疑!"按钮
   - 等待系统揭示结果
4. 如果你选择相信：
   - 直接选择你的牌并打出（相当于跳过质疑）

## 7. 基础策略指南

### 🎯 策略 1：观察对手的行为模式

**方法：**

- 仔细阅读游戏日志
- 注意对手在什么情况下会说谎
- 有些玩家喜欢早期说谎，有些则喜欢后期

** example：**

```
回合 3: 小明 声称打出了 1张 K
回合 4: 小明 声称打出了 2张 K
回合 5: 小明 声称打出了 3张 K
```

→ 小明可能真的有这么多 K 吗？值得质疑！

---

### 🎯 策略 2：适当说谎，但别太频繁

**为什么？**

- 如果你总是说真话，对手会很容易预测你
- 如果你总是说谎，对手会经常质疑你并成功

**建议：**

- 混合使用真话和谎话
- 早期可以多说谎（对手不敢轻易质疑）
- 后期少说谎（对手会更警惕）

---

### 🎯 策略 3：在关键时刻使用质疑

**什么时候质疑？**

✅ **应该质疑的情况：**

- 对手声称打出了很多张相同的牌（如 3 张 K）
- 对手在早期就声称打出大牌（如 K、Q）
- 你已经观察到对手的说谎模式

❌ **不应该质疑的情况：**

- 对手声称打出的是小牌（如 A、2）
- 这是游戏的初期，你筹码充足
- 你没有足够的信心判断

---

### 🎯 策略 4：保持筹码数量

**为什么重要？**

- 筹码为 0 = 淘汰
- 你需要保持一定的筹码来应对失误

**建议：**

- 不要过于激进地质疑（失败会失去筹码）
- 优先保护自己的筹码，而不是淘汰对手
- 当你的筹码较少时，更加保守

---

### 🎯 策略 5：利用心理战术

**方法：**

- 有时候说真话比说谎更能迷惑对手
- 故意在早期多次说谎，让对手习惯质疑你
- 然后在关键时刻说真话，让对手不敢质疑

** example：**

```
前 3 轮：你都在说谎（对手发现了）
第 4 轮：你说真话（但对手以为你还在说谎，选择质疑）
结果：对手质疑失败，失去 1 个筹码！
```

## 8. 常见错误与避免方法

### ❌ 错误 1：过于频繁地质疑

**问题：** 新手玩家经常质疑 every time，导致快速失去筹码。

**为什么是错误？**

- 质疑失败会失去筹码
- 不是每次对手都在说谎

**如何避免？**

- 只在你有信心时质疑
- 早期尽量少质疑
- 观察对手的模式后再决定

---

### ❌ 错误 2：说谎太明显

**问题：** 新手经常做出不真实的声称，容易被看穿。

** example：**

```
你的手牌: A, 2, 3, 4, 5
你声称打出了: 3张 K
```

→ 对手很容易判断你在说谎！

**如何避免？**

- 声称的牌型应该"看起来合理"
- 不要声称打出你明显没有的牌型
- 混合使用小谎话和大谎话

---

### ❌ 错误 3：不关注游戏日志

**问题：** 新手经常忽略游戏日志，无法分析对手。

**为什么是错误？**

- 游戏日志包含重要信息
- 你可以从中发现对手的模式

**如何避免？**

- 每次轮到你之前，阅读游戏日志
- 注意对手的声称是否一致
- 记录对手的说谎频率

---

### ❌ 错误 4：太早打出大牌

**问题：** 新手在早期就声称打出 K、Q 等大牌。

**为什么是错误？**

- 对手很容易判断你在说谎
- 你失去了"大牌"的可信度

**如何避免？**

- 早期声称打出小牌（A、2、3）
- 后期再声称打出大牌
- 建立"说真话"的信誉，然后突然说谎

---

### ❌ 错误 5：情绪化决策

**问题：** 连续质疑失败后，新手会变得激进或保守。

**为什么是错误？**

- 情绪化决策通常是错误的
- 你需要保持冷静和理性

**如何避免？**

- 每次决策前深呼吸
- 基于逻辑而非情绪做决定
- 如果不确定，选择保守策略

## 9. 进阶技巧

当你掌握了基础规则和策略后，可以尝试这些进阶技巧！

### 🎭 技巧 1：建立"人设"

**方法：**

- 在游戏早期建立一种"人设"
- 例如："我总是说真话"或"我总是说谎"
- 然后在关键时刻打破这种人设

** example：**

```
前 5 轮：你一直说真话（建立"诚实"人设）
第 6 轮：你说谎（对手以为你还在说真话，不质疑）
结果：你成功欺骗了对手！
```

---

### 🎭 技巧 2：反向心理

**方法：**

- 故意做出"明显"的说谎
- 对手会想："这太明显了，他不可能这么说...也许他说的是真话？"
- 然后你就成功了！

** example：**

```
你的手牌: A, A, 2, 3, 4
你声称打出了: 5张 K
```

→ 对手会想："这太假了，他不可能这么声称...也许他真的有 5 张 K？"
→ 对手选择相信
→ 你成功欺骗了对手！

---

### 🎭 技巧 3：记牌

**方法：**

- 尝试记住已经打出的牌
- 如果某种牌已经出现了很多次，对手不太可能还有
- 这样你可以更准确地判断对手是否在说谎

** example：**

```
你已经看到:
- 3 张 K 被打出
- 一副牌只有 4 张 K
→ 对手声称打出 2 张 K？很可能是在说谎！
```

---

### 🎭 技巧 4：控制游戏节奏

**方法：**

- 有时候快速决策，有时候慢慢决策
- 让对手无法从你的决策时间判断你是否说谎

** example：**

```
说真话时：假装犹豫很久
说谎时：快速决策
```

→ 对手会从你的决策时间来判断，但你反其道而行之！

---

### 🎭 技巧 5：联盟与背叛（多人游戏）

**方法：**

- 在 3-4 人游戏中，可以暂时"联盟"
- 例如：你和另一个玩家都质疑第三个玩家
- 但当只剩 2 人时，联盟就会破裂

**注意：**

- 这是高级技巧，需要经验
- 不要过于明显，否则会被针对

## 10. 常见问题解答

### ❓ Q1: 我可以随时质疑吗？

**A:** 不是的！你只能在**你的回合，且上家刚刚打出牌**时质疑。

具体来说：

- ✅ 上家打出牌后，轮到你行动时，你可以选择质疑
- ❌ 不是你的回合时，不能质疑
- ❌ 上家没有打出牌时（例如刚开局），不能质疑

---

### ❓ Q2: 如果我质疑失败，会怎样？

**A:** 你会失去 **1 个筹码**，然后轮到你打出卡牌。

---

### ❓ Q3: 如果我的筹码用完了，还能继续游戏吗？

**A:** 不能。当你的筹码降为 **0** 时，你被淘汰，不能再继续游戏。

---

### ❓ Q4: 手牌出完后会发生什么？

**A:** 当有玩家手牌出完后：

1. 所有玩家的牌都被收回
2. 重新洗牌并发牌
3. 开始新一轮

> **💡 小贴士：** 你不会出局！只是重新发牌而已。

---

### ❓ Q5: 我可以看到其他玩家的手牌吗？

**A:** 不能！你只能看到**你自己**的手牌。

这也是游戏的乐趣所在 —— 你不知道对手手里有什么牌，只能根据他们的"声称"来判断！

---

### ❓ Q6: 游戏最多支持几个玩家？

**A:** 最多 **4 个玩家**。

建议：

- **2 人** - 适合新手学习
- **3 人** - 最佳游戏体验
- **4 人** - 更复杂，更有趣

---

### ❓ Q7: 如果其他玩家长时间不行动怎么办？

**A:** 目前版本没有"超时踢出"机制。你可以：

1. 如果他们是挂机状态，建议重新开始游戏

> **💡 未来改进：** 可以添加"准备超时"机制，如果玩家长时间不准备，自动踢出房间。

---

### ❓ Q8: 我可以和远方的朋友一起玩吗？

**A:** 可以！但需要一些配置：

**方法 1 - 使用内网穿透工具（推荐）：**

1. 下载并安装 ngrok 或类似工具
2. 运行：`ngrok http 5000`
3. 复制生成的公网地址（如 `https://abc123.ngrok.io`）
4. 告诉你的朋友这个地址
5. 你的朋友在"服务器配置"中输入这个地址

**方法 2 - 使用公网服务器：**

1. 将后端部署到有公网 IP 的服务器
2. 告诉朋友服务器地址
3. 朋友在"服务器配置"中输入服务器地址

---

### ❓ Q9: 游戏结束后，如何开始新游戏？

**A:** 

1. 游戏结束后，会显示"胜利提示"弹窗
2. 点击"结束游戏并释放房间"按钮
3. 你会返回游戏大厅
4. 可以创建新游戏或加入其他游戏

---

### ❓ Q10: 为什么有时候"质疑"按钮是灰色的？

**A:** "质疑"按钮只有在以下条件满足时才可以点击：

1. ✅ 轮到你的回合
2. ✅ 上家刚刚打出牌
3. ✅ 游戏正在进行中（不是等待阶段）

如果不是你的回合，或者上家还没有打出牌，你不能质疑。
